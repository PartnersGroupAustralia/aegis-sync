# AegisSync: Enterprise Identity Validator

AegisSync is a robust corporate identity synchronization and validation suite. It provides a central Hub to manage distributed nodes, an Engine powered by Firefox Stealth (Playwright) to validate identities against various endpoints, and real-time notifications via Telegram.

## System Architecture

*   **Hub (`hub-main.js`)**: An Express server and WebSocket Hub that coordinates audits and manages connected nodes securely. Uses SQLite to persist audit logs and active node data.
*   **Engine (`engine-core.js`)**: A headless Firefox automation script using `playwright-extra` and `puppeteer-extra-plugin-stealth` to bypass basic bot detections. Routes traffic through gateways and validates identity credentials automatically.
*   **Scout (`scout-target.js`)**: Integrates with the Firecrawl API to dynamically map out login UI targets (extracting selectors for email, password, and submit buttons).
*   **Alert Manager (`alert-manager.js`)**: Dispatches real-time alerts to a configured Telegram chat upon successful identity validation.
*   **Vault & Audit**: Successful sessions (cookies) are securely persisted in the `vault/` directory, while attempts are recorded in the `audit/sync_master.db` SQLite database.

## Prerequisites

*   **Node.js**: v22+
*   **Playwright**: Firefox browser binaries
*   **Docker** (Optional, for containerized deployment)

## Setup & Installation

### Local Installation

1.  **Install Node Dependencies**:
    ```bash
    npm install
    ```
2.  **Install Playwright Browsers**:
    ```bash
    npx playwright install firefox
    ```
3.  **Configure Environment**:
    Review and update the `.env` file generated by the deployment script.
    ```dotenv
    PORT=1337
    DB_PATH=./audit/sync_master.db
    SYNC_TOKEN=<your_32_byte_hex_token>
    FIRECRAWL_API_KEY=<your_firecrawl_api_key>
    TELEGRAM_BOT_TOKEN=<your_telegram_bot_token>
    TELEGRAM_CHAT_ID=<your_telegram_chat_id>
    ```

### Docker Deployment

To run the application via Docker:
```bash
docker-compose up -d --build
```
This maps the required ports and creates persistent volumes for `./vault` and `./audit`.

## Usage

### Starting the Hub

Start the primary synchronization hub:
```bash
node hub-main.js
```
The server will begin listening on the configured `PORT` (default `1337`).

### Triggering an Audit

You can trigger a bulk identity validation via the REST API endpoint: `POST /api/perform-audit`

**Example Request:**
```bash
curl -X POST http://localhost:1337/api/perform-audit \
  -H "Content-Type: application/json" \
  -d '{
    "identities": [
      { "email": "user1@example.com", "pass": "password123" },
      { "email": "user2@example.com", "pass": "securePass!" }
    ],
    "endpoint": "https://target-portal.com/login"
  }'
```

The system processes identities in parallel batches of 5. Successful logins will:
1. Trigger a Telegram Alert.
2. Save the session context (cookies) to `./vault/user1@example.com.json`.
3. Record a `SUCCESS` entry in the `audit_log` SQLite table.

### Scouting a New Target

If you need to analyze the DOM of a new login target using Firecrawl:
```bash
node scout-target.js "https://target-portal.com/login"
```

## Security Considerations

*   **Gateways**: Place proxy IP/Ports in `gateway-nodes.txt` (one per line) to rotate the Engine's IP address.
*   **WebSocket Comms**: Node-to-Hub communications over WebSocket are secured using AES-256-GCM encryption seeded by `SYNC_TOKEN`. Keep this token strictly confidential.
*   **Vault Data**: Files inside the `vault/` directory contain active session cookies. These should be treated as highly sensitive data.

## Integration with Warp & OpenHands

This project is configured to integrate with **Warp's AI Agent (Oz)** and **OpenHands** through the Model Context Protocol (MCP).

### Running OpenHands
You can spin up an OpenHands agent in a Docker container to work within this directory:
```bash
docker run -it \
    --pull=always \
    -e SANDBOX_USER_ID=$(id -u) \
    -e WORKSPACE_BASE=$HOME/aegis-sync \
    -v $HOME/aegis-sync:/workspace \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -p 3000:3000 \
    --name openhands \
    ghcr.io/all-hands-ai/openhands:0.9
```

### Warp MCP Configuration
The repository includes a project-scoped Warp configuration file at `.warp/.mcp.json`. This allows Warp to automatically connect to both the OpenHands container and the Aegis Hub API:

```json
{
  "mcpServers": {
    "openhands": {
      "url": "http://127.0.0.1:3000/api/mcp",
      "env": {
        "OPENHANDS_API_KEY": "optional_key",
        "WORKSPACE_BASE": "/Users/jake/aegis-sync"
      }
    },
    "aegis-hub": {
      "url": "http://127.0.0.1:1337"
    }
  }
}
```
With this setup, the Warp agent can inspect the OpenHands sandbox and communicate directly with the local Aegis Hub via their respective MCP endpoints.
